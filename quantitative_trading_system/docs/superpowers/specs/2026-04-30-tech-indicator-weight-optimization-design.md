# 技术指标权重回测优化方案

## 1. 背景

当前 `TechnicalFramework` 中各技术指标的权重（RSI 15%、MACD 20%、KDJ 15%、布林带 15%、MA 25%、量比 10%）是人工设定的经验值，没有数据支撑。需要通过回测找到最优权重。

## 2. 目标

通过网格搜索，找出使**累计收益率最大化**的指标权重组合。

## 3. 回测逻辑

### 3.1 交易信号

- **买入信号**：composite ≥ 70（strong_buy）
- **空仓信号**：composite < 70

### 3.2 持仓收益计算

- 买入：次日开盘价买入
- 卖出：次日开盘价卖出
- 收益率叠加计算累计收益

### 3.3 初始资金

- 100 万人民币

## 4. 数据划分

| 阶段 | 时间范围 | 用途 |
|------|---------|------|
| 训练期 | 前 3 年 | 网格搜索找最优权重 |
| 测试期 | 后 2 年 | 验证最优权重是否有效 |

## 5. 权重搜索空间

6 个指标权重（每档 5%），之和为 100%：

| 指标 | 权重档位 |
|------|---------|
| RSI | 0%, 5%, 10%, ..., 40% |
| MACD | 0%, 5%, 10%, ..., 40% |
| KDJ | 0%, 5%, 10%, ..., 40% |
| 布林带 | 0%, 5%, 10%, ..., 40% |
| MA | 0%, 5%, 10%, ..., 40% |
| 量比 | 0%, 5%, 10%, ..., 40% |

约束：所有权重之和 = 100%。

约 `C(16,5) ≈ 4000` 种组合。

### 6.3 搜索流程

1. 枚举所有满足约束的权重组合
2. 对每组权重，遍历所有股票历史数据，逐日模拟交易
3. 计算每只股票的累计收益率
4. 汇总所有股票的加权平均收益率（或中位数）
5. 选择收益率最高的权重组合

## 7. 输出

1. **最优权重组合**：6 个指标的具体权重值
2. **训练期收益**：最优权重在训练期的累计收益率
3. **测试期收益**：最优权重在测试期的累计收益率
4. **基准对比**：等权重（各 16.67%）的收益率作为基准

## 8. 约束

- 单进程，不使用多进程/多线程（避免系统卡死）
- 每次只加载必要的数据，避免内存溢出
