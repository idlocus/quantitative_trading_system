# 全A股深度学习训练方案设计

## 1. 概述

**目标**：使用 Wind 数据库获取全市场所有股票（~6000只），以最近5年历史行情进行 LSTM 深度学习训练，预测次日股价走势（三分类：涨/跌/震荡）。

**核心策略**：增量滚动训练（Incremental Rolling Training），通过分批加载数据、控制检查点、渐进式学习，在资源受限环境下完成全量训练。

---

## 2. 训练参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 每批股票数 | 500只 | 平衡内存占用和多样性 |
| 总批次数 | 12批 | 12 × 500 = 6000只覆盖全市场 |
| 历史数据窗口 | 5年 (~1250交易日) | 从Wind数据库获取 |
| 序列长度 | 60天 | LSTM输入窗口 |
| 涨跌阈值 | ±1% | >1%为涨，<-1%为跌，中间为震荡 |
| 训练轮次 | 30 epochs/批 | 支持早停 patience=5 |
| 早停策略 | patience=5 | 验证集5轮无改善则停止 |
| 学习率 | 0.001 | ReduceLROnPlateau调度 |
| 批量大小 | 32 | 训练batch_size |

---

## 3. 模型架构

**LSTM模型**（来自现有 `lstm_predictor.py`）:
- 输入: `(batch_size, seq_len=60, n_features)`
- 结构: 双层LSTM (hidden=128, layers=2, dropout=0.3)
- 输出: `(batch_size, 3)` - 涨/跌/震荡概率

**特征工程**（来自现有 `preprocessor.py`）:
- 价格特征: 收盘/开盘比、高低比、量比
- 均线特征: 5/10/20/60日均线偏离度
- 波动率: 20日/60日收益率标准差
- 动量: 5/10/20日收益率
- 相对位置: 价格在N日高低区间的位置

---

## 4. 训练流程

```
┌─────────────────────────────────────────────────────────────┐
│                      训练主流程                               │
├─────────────────────────────────────────────────────────────┤
│  Step 1: 连接Wind数据库 (oracledb)                           │
│  Step 2: 获取全量A股列表 (~6000只)                            │
│  Step 3: 随机打乱，分成12批，每批500只                        │
│  Step 4: 循环训练12批:                                       │
│           ┌─────────────────────────────────────────────┐    │
│           │  4.1 加载该批500只股票的5年OHLCV数据          │    │
│           │  4.2 构建特征DataFrame (约20个特征)           │    │
│           │  4.3 创建序列样本 (60日窗口)                   │    │
│           │  4.4 创建标签 (±1%阈值三分类)                 │    │
│           │  4.5 划分训练70%/验证10%/测试20%             │    │
│           │  4.6 训练LSTM (30epochs, 早停)               │    │
│           │  4.7 保存检查点 lstm_wind_batch_{n}.pt       │    │
│           │  4.8 释放该批数据内存 → 进入下一批             │    │
│           └─────────────────────────────────────────────┘    │
│  Step 5: 输出最终模型 lstm_wind_final.pt                      │
│  Step 6: 生成训练报告 metrics_wind_final.json                 │
└─────────────────────────────────────────────────────────────┘
```

---

## 5. 资源控制措施

| 措施 | 实现 |
|------|------|
| 内存隔离 | 每批训练后显式 `del symbol_data`, `gc.collect()` |
| 检查点保存 | 每批训练完成保存 `.pt` 文件 |
| 早停机制 | `patience=5` 验证集无改善则停止该批 |
| 梯度裁剪 | `clip_grad_norm_(max_norm=1.0)` |
| 数据校验 | 跳过OHLCV数据不足100天的股票 |
| 进度日志 | 每50只股票报告加载进度 |

## 7. 预测报告生成

训练完成后，使用 `run_dl_wind_all_a.py` 生成全A股预测报告:

```

```

**报告内容**:
- 全市场信号分布统计
- TOP30推荐股票
- 每只股票的DL预测、置信度、综合得分

## 8. 预计资源占用

| 资源 | 预计占用 |
|------|----------|
| 单批内存 | ~2-3 GB (500只 × 1250天 × 5字段) |
| GPU显存 | ~1-2 GB (batch=32, hidden=128) |
| 单批训练时间 | ~10-20分钟 (GPU) / ~30-60分钟 (CPU) |
| 总训练时间 | ~2-4小时 |
| 磁盘占用 | ~500 MB (12检查点 + 历史) |

---

## 9. 恢复机制

如果训练过程崩溃:
1. 检查已保存的 `lstm_wind_batch_*.pt` 文件
2. 从最后一个有效检查点恢复
3. 跳过已完成的批次继续训练
